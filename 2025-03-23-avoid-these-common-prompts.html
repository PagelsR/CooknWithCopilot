Avoid These Common Prompts

Posted on March 23, 2025

Copilot is powerful, but like any assistant, it performs best with clear instructions. Some prompt styles can trip it up or lead to irrelevant code. Let’s fix that.

❌ Too Vague

A prompt like // write code gives Copilot almost nothing to work with. Be specific about what you want — think input, output, and purpose.

❌ Too Many Tasks in One Prompt

“Create a function to sort and then filter a list and save it to a file” — whoa, slow down! Break that into multiple prompts or comments. It’ll handle each one more accurately.

❌ Missing Context

Copilot works best with the full picture. If you’re prompting it without any surrounding code or types, the suggestions may not align with your actual app or language.

✅ Better Prompting Ideas

  • // returns true if the email is valid
  • // fetch user data from API and cache the result
  • // write a Jest test for this function

Clear and focused prompts = better results, less editing, and faster dev time. Clean prompts are king.
